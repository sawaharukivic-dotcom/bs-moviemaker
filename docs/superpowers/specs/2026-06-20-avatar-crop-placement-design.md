# アバターのトリミング＆自由配置＋音声ガイド 設計メモ

作成日: 2026-06-20 / 対象: bs-moviemaker

本メモは独立した 2 機能を扱う（同じ実装ラウンドで投入）:
1. アバターの空間トリミング＆自由配置（メイン）
2. 音声ガイド＋無音検知（録画マイク ON の周知・無音素材の事前警告）

## 背景・目的
キャストが iPhone の画面録画で素材を撮ると、画面上部（カメラ／ステータスバー付近）に
**録画中マーク（赤丸など）**が写り込む。また、フル画面録画（縦長 9:19.5 等）を 9:16 に
`contain` で収めると左右に黒帯（ピラーボックス）が出て小さく見える。

→ **ソース動画の好きな範囲を切り抜き（録画マーク・帯を除外）、9:16 キャンバス内に自由配置**
できるようにする。

現状（`compositor.js:93-104` `drawAvatar`）はソース全体を `containRect` で出力中央に転写するのみ。
時間トリミング（尺 in/out）は `timeline.js` に既存。本件は**空間トリミング＋配置**の追加。

## モデル：2つの正規化矩形
- **cropRect**（ソース座標 0..1）= ソース動画のどの部分を使うか。既定 `{x:0,y:0,w:1,h:1}`（全体）。
- **placeRect**（出力座標 0..1）= 切り抜き結果を 9:16 キャンバスのどこに置くか。既定は中央 `contain` フィット。

キャンバスは **9:16 固定**。出力解像度に依存しないよう座標はすべて正規化。

### 配置のアスペクト＝比率維持
配置は「位置＋サイズ（縦横比固定）」。非等倍ストレッチは禁止（アバターが歪むため）。
placeRect は cropRect の実アスペクトを保つよう導出（移動＋等倍拡縮のみ許可）。

## 合成の変更（compositor.js）
新インターフェース（純データ・DOM 非依存を維持）:
```
comp.setAvatarTransform({ crop:{x,y,w,h}, place:{x,y,w,h} });
```
`drawAvatar` の keyCanvas→出力転写を crop/place 指定の `drawImage` に変更:
```
ctx.drawImage(keyCanvas,
  crop.x*keyW, crop.y*keyH, crop.w*keyW, crop.h*keyH,   // ソース部分矩形
  place.x*W,   place.y*H,   place.w*W,   place.h*H);     // 9:16 内の配置先
```
- crop/place 未設定時は現状互換（crop=全体, place=contain フィット）を維持。
- chromakey シェーダは不変。crop は緑抜き後の keyCanvas に対して適用。

## 操作（UI）
CLAUDE.md 方針どおり、ドラッグ操作の実装は **`front/index.html` のインライン JS**に置く
（`js/*.js` モジュールは DOM を触らない）。

1. **切り抜き枠**：ソースの一時停止フレーム上にドラッグ枠＋四隅ハンドルを重ね、cropRect を決める。
2. **配置**：9:16 プレビュー上で、切り抜いたアバターをドラッグ移動＋四隅で拡縮し、placeRect を決める。

両者は「ある要素の上で正規化矩形をドラッグ編集する」同一部品なので、**1 つの再利用ヘルパー**を
2 回使う（タッチ／マウス両対応・四隅ハンドル・最小サイズ制約）。出力は正規化 rect。

## 入出力契約（まとめ）
| 単位 | 何をする | 入力 | 出力 |
|---|---|---|---|
| compositor.setAvatarTransform | crop/place を保持し drawAvatar に反映 | `{crop, place}` 正規化 rect | （なし・描画に反映） |
| rect-editor（index.html 内） | 要素上で矩形をドラッグ編集 | 対象要素・初期 rect・アスペクト固定可否 | 変更時に正規化 rect を callback |

依存方向は不変（`ui → compositor → chromakey`）。

## WYSIWYG / 検証
- プレビュー（`renderFrame`）と書き出し（`playRange` 中の録画）は同じ `compositor` を通すため、
  crop/place は両方に自動反映＝プレビューと出力が一致する。
- 実機（iOS Safari / Android Chrome）でのタッチ操作・書き出し一致は**実機検証が必要**。

## 機能2：音声ガイド＋無音検知

### 背景
iPhone の画面録画は**デフォルトでマイク OFF**のため無音になりがち。コントロールセンターの
録画ボタン長押し → マイク ON で声が動画に入る。動画に音声があれば既存 `audiomixer.connectVoice(el)`
がそのまま声を使うので、**マイク ON で録ってもらえれば追加実装なしで声が乗る**。本機能は
「録り方の周知」と「無音素材を作る前に気づかせる」ことが目的（在アプリ内の声録りはしない＝B/C は非対象）。

### ガイド表示
アップロード画面に短い手順を常時表示：「画面録画はマイク OFF だと無音。録画ボタン長押し →
マイク ON で録ってね」。静的 UI テキスト（`front/index.html`）。

### 無音検知（ベストエフォート）
動画ロード後、音声トラックの有無を**最善努力で**判定し、無音らしければ非ブロッキングの
警告バナーを出す（「音声が検出できません。録画時にマイクを ON にしましたか？」）。作成は止めない
（誤検知＝false negative の可能性があるため）。

判定ヒューリスティック（取れるものから順に。確実な単一手段は無い）:
- `videoEl.mozHasAudio` / `videoEl.audioTracks?.length`（対応ブラウザ）
- `videoEl.webkitAudioDecodedByteCount`（再生後に増加するか）
- 上記が不定なら WebAudio で短時間プローブ（AnalyserNode で信号の有無）

audiomixer に best-effort 判定 `probeAudio(el): Promise<'yes'|'no'|'unknown'>` を追加（音声の
責務はオーディオモジュールが持つ）。バナー表示は UI（index.html）。`'unknown'` のときは警告を出さない
か、控えめな注意に留める。

> **iOS Safari では音声トラック検出が不安定**で、コードからは断定不可。判定はあくまで補助であり、
> ガイド表示が主。判定の信頼度は**実機検証が必要**。

## 非対象（YAGNI）
- 非等倍ストレッチ配置、回転、複数アバター、切り抜きアニメーション。
- 録画マーク自動検出（手動切り抜きで対応）。
- アプリ内での声の後録り（B）／別音声ファイル合流（C）。無音検知はブロックしない。

## MVP 合格条件
- 機能1: ソース動画の上部（録画マーク）を切り抜きで除外でき、切り抜いた結果を 9:16 枠内で自由に
  移動・拡縮（比率維持）でき、その配置どおりに 9:16 動画が書き出せる。
- 機能2: アップロード画面にマイク ON のガイドが出る。音声のある動画は従来どおり声が乗る。
  無音らしい動画では警告バナーが出る（作成はブロックしない）。
